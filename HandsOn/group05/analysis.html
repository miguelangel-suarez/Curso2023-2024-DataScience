Data Source Analysis

Los campos de información proporcionados y su explicación son los siguientes:

1. MES : corresponde al mes en el que se realizan los distintos servicios especificados en la fila correspondiente

DATA TYPE: STRING.       RANGE: [ENERO- DICIEMBRE]

2. LOTE: se recoge el lote del Contrato del Servicio de Limpieza de los Espacios Públicos de Madrid en el que se han realizado los servicios especificados en la fila correspondiente.

DATA TYPE: INT  RANGE: [1-6]

3. CÓDIGO DISTRITO: Número de distrito al que se encuentran asignados los datos de los servicios de la fila correspondiente (del 1 al 21).

DATA TYPE: INT RANGE: [1-21]

4. DISTRITO: Distrito al que se encuentran asignados los datos de los servicios de la fila correspondiente.

DATA TYPE: STR

5. RECOGIDA DE MUEBLES 010: se recoge el pesaje en Kg. de los muebles y enseres recogidos a través del servicio del Ayuntamiento de Madrid de Recogida gratuita de muebles y enseres, solicitados a través del servicio 010.

DATA TYPE:  INT   RANGE: [100-108.600]

6. RECOGIDA DE MUEBLES PROGRAMADA: se recoge el pesaje en Kg. de los muebles y enseres recogidos a través del servicio de recogida programada del Ayuntamiento de Madrid de Recogida gratuita de muebles y enseres programada 1 vez al mes.

DATA TYPE: INT    RANGE: [0-59.880]

7. RECOGIDA DE MUEBLES ABANDONADOS: se recoge el pesaje en Kg. de los muebles y enseres abandonados en calle por los ciudadanos, sin haberse acogido a ninguno de los Servicios ofrecidos por el Ayuntamiento de Madrid.

DATA TYPE: INT    RANGE: [0-255.200]

8. RECOGIDA DE MUEBLES - TOTAL: se recoge el pesaje en Kg. de los muebles y enseres totales recogidos en calle (Suma de las columnas 4, 5 y 6).

DATA TYPE: INT    RANGE: [440-275.369]

9. RECOGIDA DE RESIDUOS VIARIOS  SEPARACIÓN RESTO: se recoge el pesaje en Kg. de los residuos que se recogen por los Servicios de Limpieza del Ayuntamiento de Madrid y son susceptibles de separación en las Plantas de Reciclaje Municipales por su composición (especialmente residuos de papeleras de resto situadas en vía pública).

DATA TYPE: INT RANGE: [0-255.200]

10. RECOGIDA DE RESIDUOS VIARIOS  ORGÁNICA: se recoge el pesaje en Kg. De los residuos orgánicos (ramas y hojas) recogidos por los Servicios de Limpieza del Ayuntamiento de Madrid.

DATA TYPE: INT RANGE: [0-169.019]

11. RECOGIDA DE RESIDUOS VIARIOS - ENVASES: se recoge el pesaje en Kg. De los residuos de envases que se recogen por los Servicios de Limpieza del Ayuntamiento de Madrid, procedentes principalmente de las papeleras de envases situadas en vía pública.

DATA TYPE: INT   RANGE: [0-47.800]

12. RECOGIDA DE RESIDUOS VIARIOS  PAPEL Y CARTÓN: se recoge el pesaje en Kg. de los residuos de papel y cartón que se han abandonado en la vía pública sin ningún tipo de contenerización.

DATA TYPE: INT RANGE: [0-38.990]

13. RECOGIDA DE RESIDUOS VIARIOS  RCDs/VARIOS: se recoge el pesaje en Kg. de los residuos que se han abandonado en la vía pública sin ningún tipo de contenerización (bolsas de basura domiciliaria, escombros esparcidos, etc) y que por su presentación no son susceptibles de separación.

DATA TYPE: INT   RANGE: [0-786.480]

14. RECOGIDA DE RESIDUOS VIARIOS  TOTAL: se recoge el pesaje en Kg. de la totalidad de residuos recogidos. (Suma de las columnas 9, 10, 11, 12 y 13).

DATA TYPE:  INT   RANGE: [7.932-910.700]

15. BARRIDO MANUAL: se recoge el número de barridos manuales realizados en el distrito y mes. El barrido manual es el que realiza el personal con carro portacontenedores, cepillo, escobijo, pala y cualquier herramienta auxiliar necesaria para el desarrollo de su trabajo.

DATA TYPE: INT   RANGE: [402-1.778]

16. BARRIDO MANTENIMIENTO: se recoge el número de barridos de mantenimiento realizados en el distrito y mes. El barrido de mantenimiento es un barrido manual, que se realiza de forma complementaria a otro servicio (barrido o baldeo) realizado a lo largo de la jornada, por lo que es muy rápido y con gran rendimiento.

DATA TYPE: INT   RANGE: [0-1.895]

17. BARRIDO MECÁNICO: recoge el número de barridos mecánicos realizados en el distrito y mes. El barrido mecánico es el que se realiza con una máquina barredora y el personal suficiente para la realización del servicio.

DATA TYPE: INT   RANGE: [0-193]

18. BARRIDO MIXTO: recoge el número de barridos mixtos realizados en el distrito y mes. El barrido mixto es el que se realiza con una máquina barredora y el personal suficiente dotado de sopladoras para la realización del servicio.

DATA TYPE: INT   RANGE: [14-468]

19. BALDEO MECÁNICO: recoge el número de baldeos mecánicos realizados en el distrito y mes. El baldeo mecánico es el que se realiza con un camión cisterna dotado de equipo de baldeo mecánico.

DATA TYPE: INT   RANGE: [0-35]

20. BALDEO MIXTO: recoge el número de baldeos mixtos realizados en el distrito y mes. El baldeo mixto es el que se realiza con un camión cisterna dotado de equipo de baldeo mixto junto con el personal suficiente para la realización del servicio.

DATA TYPE:  INT RANGE: [78-822]

21. HIDROLIMPIADOR: recoge el número de servicios realizados con equipo para eliminación de manchas. Este servicio se realiza con vehículo hidrolimpiador dotado de los equipos necesarios para la eliminación de manchas y el personal suficiente para la realización del servicio.

DATA TYPE: INT RANGE: [0-407]

22. ELIMINACIÓN DE GRAFITIS: recoge el número de grafitis eliminados en el distrito y mes. El servicio de eliminación de grafitis y pintadas es el que se realiza con vehículo quitapintadas (dotado de los equipos necesarios para la eliminación de éstas) y el personal suficiente para la realización del servicio.

DATA TYPE: INT   RANGE: [0-1.377]

23. ELIMINACIÓN DE GRAFITIS - M2: recoge la superficie en m2 de grafitis eliminados en el distrito y mes.

DATA TYPE: INT RANGE: [0-16.016]

24. BOLSAS CANINAS: recoge el número de bolsas para la recogida de excrementos caninos suministradas a través de los expendedores situados en algunas de las papeleras de la ciudad.

DATA TYPE: INT RANGE: [177.600-906.430]

Resource name strategy

Con el propósito de hacer nuestra antología, teneos que definir una estrategia de nombrado con la que asignar URIs apropiadas para los elementos de nuestra base de datos. Hemos definido dos namespace principales. Uno para las clases principales que aparecen en nuestra base de datos y otra para las nombrar las relaciones existentes entre dichas clases.

·        http://madrid.limpiezaurbana.es/classes# Utilizaremos esta URI para las clases

·        http://madrid.limpiezaurbana.es/properties# Utilizaremos esta URI para las relaciones

Cada atributo de las clases será identificado por una URI única, añadiendo al final de esa URI el nombre específico de esa propiedad. Cuando se trata de las instancias o ejemplares, se utilizará "rdf:type" para definirlas como ejemplos pertenecientes a una clase dentro del namespace de esa clase en particular.
